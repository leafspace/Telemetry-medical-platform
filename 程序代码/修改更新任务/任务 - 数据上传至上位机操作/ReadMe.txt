项目简介：
    本项目是一个基于树莓派的嵌入式医疗语音识别|图像识别系统。

    医疗嵌入式医嘱信息录入系统：
        用户打开软件系统，通过录音设备开始说话：语句格式如：[304] 号病人 [输液|打针|其余] [葡萄糖|其他] [5] [毫升|毫克].
        树莓派系统将录入的信息识别出结果并将录音数据和结果数据上传至上位机。
        上位机将获取的信息整合，录入数据库，并刷新WEB管理系统。

    医疗嵌入式呼吸机参数识别系统：
        用户打开软件系统，通过对摄像头照射区域的选取，将选区内的数据识别，并上传至上位机。
        同时，上位机能够实时观看当前录像。

    

    当前难点：
        将结果上传至上位机时：当传输格式固定的大量数据时（本地，将数据封装成对象传输），必然使用socket传输。在系统中嵌入socket线程（作为客户端，web端为服务器端），通过socket通信。

        问题：
            1）后台的单独线程接收到数据之后如何将数据实时刷新显示到web页面。
            2）在大数据量情况下（视频状态下，每秒传输MB级别的数据）如何提高传输速度。

            3）现有情况下是否存在比socket更好的传输方式。
            4）如果使用socket，那么在web端应该如何操作（注明：现实现代码中，在session监听器中，创建session时，运行java socket线程。但是，传输时无法进行跳转等操作）
